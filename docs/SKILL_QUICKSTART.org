* Quick Start

Want to create your first Crichton WASM skill? This guide gets you started in 5 minutes.

** 1. Install Compilation Tools

#+begin_src bash
# Debian/Ubuntu
sudo apt install wabt

# macOS
brew install wabt
#+end_src

** 2. Create Skill Directory

#+begin_src bash
mkdir -p ~/.crichton/skills/my-skill
cd ~/.crichton/skills/my-skill
#+end_src

** 3. Create Manifest (skill.toml)

#+begin_src toml
[skill]
name = "my-skill"
version = "1.0.0"
author = "you@example.com"
description = "My first Crichton skill"
signature = ""

[capabilities]
http_domains = []
max_memory_mb = 64
max_cpu_seconds = 30
#+end_src

** 4. Write Skill Code (skill.wat)

#+begin_src wasm
(module
  ;; Import Crichton's log function
  (import "env" "log" (func $log (param i32 i32 i32)))

  ;; Export memory (required)
  (memory (export "memory") 1)

  ;; Your message
  (data (i32.const 0) "Hello from my skill!")

  ;; Main function
  (func (export "main") (result i32)
    ;; Log message: log(INFO, offset, length)
    (call $log
      (i32.const 1)   ;; INFO level
      (i32.const 0)   ;; message at offset 0
      (i32.const 19)) ;; message length

    ;; Return result
    (i32.const 123)))
#+end_src

** 5. Compile to WASM

#+begin_src bash
wat2wasm skill.wat -o skill.wasm
#+end_src

** 6. Test Your Skill

#+begin_src lisp
;; From SLIME/SLY REPL connected to Crichton daemon
(let* ((wasm-path #p"~/.crichton/skills/my-skill/skill.wasm")
       (wasm-bytes (with-open-file (s wasm-path :element-type '(unsigned-byte 8))
                     (let ((bytes (make-array (file-length s) :element-type '(unsigned-byte 8))))
                       (read-sequence bytes s)
                       bytes))))
  (crichton/wasm:run-wasm-bytes-with-host-fns wasm-bytes "main"))
;; => 123
;; (also logs: "Hello from my skill!")
#+end_src

* Troubleshooting

** "wat2wasm: command not found"

Install the WebAssembly Binary Toolkit (WABT):

#+begin_src bash
sudo apt install wabt  # Debian/Ubuntu
brew install wabt      # macOS
#+end_src

** "Module compile error"

Check your WAT syntax:

#+begin_src bash
wat2wasm --debug-names skill.wat
#+end_src

** "Export 'memory' not found"

WASM skills must export their linear memory:

#+begin_src wasm
(memory (export "memory") 1)
#+end_src

** "Export 'main' not found"

Your skill must export at least one function:

#+begin_src wasm
(func (export "main") (result i32)
  (i32.const 42))
#+end_src
