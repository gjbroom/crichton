* Overview

The RSS Filter skill is a pure function that receives RSS items from Crichton's
built-in RSS reader and returns only those items matching user-specified keywords.

** Design Philosophy

- *Pure function* — No side effects, no network access, no persistent state
- *Minimal capabilities* — Only logging (for debugging during development)
- *Daemon orchestrates* — The built-in RSS reader fetches feeds, invokes this skill
  for filtering, and handles notifications separately

** Example Flow

#+begin_example
1. Daemon's RSS reader fetches "CBC News" feed (built-in, trusted code)
2. Daemon invokes rss-filter skill with items + keywords
3. Skill returns matching items (pure computation)
4. Daemon decides what to do with matches (notify via Pushover, log, etc.)
#+end_example

* Interface

** Function: =filter_items=

Filter a list of RSS items by keywords.

*** Parameters

| Name          | Type     | Required | Default               | Description                              |
|---------------+----------+----------+-----------------------+------------------------------------------|
| items         | array    | yes      | —                     | RSS items from daemon's reader           |
| keywords      | array    | yes      | —                     | Keywords/phrases to match                |
| match_mode    | string   | no       | "any"                 | "any" (OR) or "all" (AND)                |
| search_fields | array    | no       | ["title","description"] | Fields to search                         |
| case_sensitive| boolean  | no       | false                 | Case-sensitive matching                  |

* Daemon Integration

** RSS Reader Task Configuration

The daemon's RSS reader would be configured with optional filters:

#+begin_src lisp
;; Example: User creates a filtered RSS task
(defun create-rss-filter-task (name feed-url keywords &key 
                               (interval 900)  ; 15 minutes
                               (match-mode "any")
                               (notify t))
  "Create a scheduled task that fetches RSS and filters by keywords"
  (schedule-task
    :name (format nil "rss:~A" name)
    :interval interval
    :function (lambda ()
                (let* ((items (fetch-rss-feed feed-url))
                       (result (invoke-skill "rss-filter" "filter_items"
                                 (list :items items
                                       :keywords keywords
                                       :match-mode match-mode)))
                       (matches (getf result :matches)))
                  (when (and notify matches)
                    (notify-rss-matches name matches))))))

;; Example usage
(create-rss-filter-task 
  "carney-watch"
  "https://www.cbc.ca/webfeed/rss/rss-topstories"
  '("Carney" "Bank of Canada" "interest rate")
  :match-mode "any"
  :notify t)
#+end_src

** Notification Handling (Daemon Side)

The daemon decides how to handle matches, keeping notification logic out of the skill:

#+begin_src lisp
(defun notify-rss-matches (filter-name matches)
  "Send notifications for RSS filter matches"
  (dolist (item matches)
    (invoke-skill "pushover" "send_notification"
      (list :title (format nil "[~A] ~A" filter-name (getf item :title))
            :message (truncate-string (getf item :description) 200)
            :url (getf item :link)
            :priority 0))))
#+end_src

** Deduplication (Daemon Side)

Since the skill is pure, deduplication happens in the daemon:

#+begin_src lisp
(defvar *rss-seen-items* (make-hash-table :test 'equal)
  "Track seen RSS item IDs to avoid duplicate notifications")

(defun filter-unseen-items (items)
  "Remove items that have already been seen"
  (loop for item in items
        for id = (getf item :id)
        unless (gethash id *rss-seen-items*)
        collect item
        do (setf (gethash id *rss-seen-items*) t)))

;; In the RSS task:
(let* ((items (fetch-rss-feed feed-url))
       (unseen (filter-unseen-items items))  ; Daemon deduplicates
       (result (invoke-skill "rss-filter" "filter_items"
                 (list :items unseen
                       :keywords keywords))))
  ...)

* Implementation

** Rust WASM Implementation

#+begin_src rust
// src/lib.rs
//
// RSS Filter Skill - Pure function implementation

#![no_std]

extern crate alloc;

use alloc::string::String;
use alloc::vec::Vec;
use alloc::format;
use serde::{Deserialize, Serialize};

// Host function imports (logging only)
extern "C" {
    fn log_debug(msg_ptr: *const u8, msg_len: usize);
}

fn log(msg: &str) {
    unsafe {
        log_debug(msg.as_ptr(), msg.len());
    }
}

// Input types
#[derive(Deserialize)]
struct FilterParams {
    items: Vec<RssItem>,
    keywords: Vec<String>,
    #[serde(default = "default_match_mode")]
    match_mode: String,
    #[serde(default = "default_search_fields")]
    search_fields: Vec<String>,
    #[serde(default)]
    case_sensitive: bool,
}

#[derive(Deserialize, Clone)]
struct RssItem {
    id: String,
    title: String,
    #[serde(default)]
    description: String,
    #[serde(default)]
    content: String,
    #[serde(default)]
    link: String,
    #[serde(default)]
    published: String,
    #[serde(default)]
    feed_name: String,
}

fn default_match_mode() -> String {
    String::from("any")
}

fn default_search_fields() -> Vec<String> {
    vec![String::from("title"), String::from("description")]
}

// Output types
#[derive(Serialize)]
struct FilterResult {
    matches: Vec<MatchedItem>,
    statistics: Statistics,
}

#[derive(Serialize)]
struct MatchedItem {
    id: String,
    title: String,
    description: String,
    link: String,
    published: String,
    feed_name: String,
    matched_keywords: Vec<String>,
}

#[derive(Serialize)]
struct Statistics {
    items_scanned: usize,
    items_matched: usize,
}

// Main entry point
#[no_mangle]
pub extern "C" fn filter_items(params_ptr: *const u8, params_len: usize,
                                out_ptr: *mut u8, out_len: usize) -> i32 {
    // Parse input
    let params_slice = unsafe { 
        core::slice::from_raw_parts(params_ptr, params_len) 
    };
    
    let params: FilterParams = match serde_json::from_slice(params_slice) {
        Ok(p) => p,
        Err(e) => {
            log(&format!("Failed to parse params: {}", e));
            return -1;
        }
    };
    
    log(&format!("Filtering {} items with {} keywords", 
                 params.items.len(), params.keywords.len()));
    
    // Process each item
    let mut matches = Vec::new();
    
    for item in &params.items {
        let matched_keywords = check_item_match(
            item,
            &params.keywords,
            &params.match_mode,
            &params.search_fields,
            params.case_sensitive,
        );
        
        if !matched_keywords.is_empty() {
            matches.push(MatchedItem {
                id: item.id.clone(),
                title: item.title.clone(),
                description: item.description.clone(),
                link: item.link.clone(),
                published: item.published.clone(),
                feed_name: item.feed_name.clone(),
                matched_keywords,
            });
        }
    }
    
    log(&format!("Found {} matches", matches.len()));
    
    // Build result
    let result = FilterResult {
        statistics: Statistics {
            items_scanned: params.items.len(),
            items_matched: matches.len(),
        },
        matches,
    };
    
    // Serialize output
    let output = match serde_json::to_vec(&result) {
        Ok(v) => v,
        Err(e) => {
            log(&format!("Failed to serialize result: {}", e));
            return -2;
        }
    };
    
    if output.len() > out_len {
        log("Output buffer too small");
        return -3;
    }
    
    unsafe {
        core::ptr::copy_nonoverlapping(output.as_ptr(), out_ptr, output.len());
    }
    
    output.len() as i32
}

fn check_item_match(
    item: &RssItem,
    keywords: &[String],
    match_mode: &str,
    search_fields: &[String],
    case_sensitive: bool,
) -> Vec<String> {
    // Build searchable text from specified fields
    let mut searchable = String::new();
    
    for field in search_fields {
        let field_value = match field.as_str() {
            "title" => &item.title,
            "description" => &item.description,
            "content" => &item.content,
            _ => continue,
        };
        searchable.push_str(field_value);
        searchable.push(' ');
    }
    
    // Normalize case if needed
    let searchable = if case_sensitive {
        searchable
    } else {
        searchable.to_lowercase()
    };
    
    // Find matching keywords
    let mut matched = Vec::new();
    
    for keyword in keywords {
        let needle = if case_sensitive {
            keyword.clone()
        } else {
            keyword.to_lowercase()
        };
        
        if searchable.contains(&needle) {
            matched.push(keyword.clone());
        }
    }
    
    // Apply match mode
    match match_mode {
        "all" => {
            if matched.len() == keywords.len() {
                matched
            } else {
                Vec::new()
            }
        }
        _ => matched, // "any" mode - return whatever matched
    }
}

// Memory allocation for WASM
#[no_mangle]
pub extern "C" fn alloc(size: usize) -> *mut u8 {
    let mut buf = Vec::with_capacity(size);
    let ptr = buf.as_mut_ptr();
    core::mem::forget(buf);
    ptr
}

#[no_mangle]
pub extern "C" fn dealloc(ptr: *mut u8, size: usize) {
    unsafe {
        let _ = Vec::from_raw_parts(ptr, 0, size);
    }
}

#[cfg(test)]
mod tests {
    use super::*;
    
    fn make_item(id: &str, title: &str, desc: &str) -> RssItem {
        RssItem {
            id: id.to_string(),
            title: title.to_string(),
            description: desc.to_string(),
            content: String::new(),
            link: format!("https://example.com/{}", id),
            published: String::new(),
            feed_name: "Test Feed".to_string(),
        }

    #[test]
    fn test_any_match() {
        let item = make_item("1", "Carney announces rate decision", 
                            "Bank of Canada governor speaks");
        let keywords = vec!["Carney".to_string(), "Trudeau".to_string()];
        
        let matched = check_item_match(
            &item, &keywords, "any", 
            &["title".to_string()], false
        );
        
        assert_eq!(matched, vec!["Carney"]);
    }
    
    #[test]
    fn test_all_match() {
        let item = make_item("1", "Carney and Trudeau meet", "Discussion of rates");
        let keywords = vec!["Carney".to_string(), "Trudeau".to_string()];
        
        let matched = check_item_match(
            &item, &keywords, "all",
            &["title".to_string()], false
        );
        
        assert_eq!(matched.len(), 2);
    }
    
    #[test]
    fn test_all_match_fails() {
        let item = make_item("1", "Carney speaks", "No mention of PM");
        let keywords = vec!["Carney".to_string(), "Trudeau".to_string()];
        
        let matched = check_item_match(
            &item, &keywords, "all",
            &["title".to_string()], false
        );
        
        assert!(matched.is_empty());
    }
    
    #[test]
    fn test_case_insensitive() {
        let item = make_item("1", "CARNEY SPEAKS", "Important news");
        let keywords = vec!["carney".to_string()];
        
        let matched = check_item_match(
            &item, &keywords, "any",
            &["title".to_string()], false
        );
        
        assert_eq!(matched, vec!["carney"]);
    }
    
    #[test]
    fn test_case_sensitive() {
        let item = make_item("1", "CARNEY SPEAKS", "Important news");
        let keywords = vec!["carney".to_string()];
        
        let matched = check_item_match(
            &item, &keywords, "any",
            &["title".to_string()], true  // case sensitive
        );
        
        assert!(matched.is_empty());
    }
    
    #[test]
    fn test_multiple_fields() {
        let item = make_item("1", "Economic news", "Carney discusses policy");
        let keywords = vec!["Carney".to_string()];
        
        // Title only - should not match
        let matched = check_item_match(
            &item, &keywords, "any",
            &["title".to_string()], false
        );
        assert!(matched.is_empty());
        
        // Title + description - should match
        let matched = check_item_match(
            &item, &keywords, "any",
            &["title".to_string(), "description".to_string()], false
        );
        assert_eq!(matched, vec!["Carney"]);
    }

* Security Analysis

** Minimal Attack Surface

As a pure function with no capabilities beyond logging:

| Threat                    | Mitigated? | Notes                                    |
|---------------------------+------------+------------------------------------------|
| Network exfiltration      | ✓          | No network capability                    |
| Filesystem access         | ✓          | No filesystem capability                 |
| State manipulation        | ✓          | No storage capability                    |
| Privilege escalation      | ✓          | No IPC capability                        |
| Resource exhaustion (CPU) | Partial    | Daemon enforces execution time limits    |
| Resource exhaustion (mem) | Partial    | WASM memory limits enforced by runtime   |

** Input Validation

The skill should validate inputs to prevent DoS:

#+begin_src rust
const MAX_ITEMS: usize = 1000;
const MAX_KEYWORDS: usize = 100;
const MAX_KEYWORD_LEN: usize = 200;
const MAX_ITEM_TEXT_LEN: usize = 100_000;

fn validate_params(params: &FilterParams) -> Result<(), &'static str> {
    if params.items.len() > MAX_ITEMS {
        return Err("Too many items");
    }
    if params.keywords.len() > MAX_KEYWORDS {
        return Err("Too many keywords");
    }
    for kw in &params.keywords {
        if kw.len() > MAX_KEYWORD_LEN {
            return Err("Keyword too long");
        }
    }
    for item in &params.items {
        let total_len = item.title.len() + item.description.len() + item.content.len();
        if total_len > MAX_ITEM_TEXT_LEN {
            return Err("Item text too long");
        }
    }
    Ok(())
}

* Summary

The RSS Filter skill is intentionally minimal:

- *Pure function* — Input items, output matches
- *No side effects* — Daemon handles fetch, notify, dedupe
- *Minimal capabilities* — Logging only (removable)
- *Easy to audit* — ~200 lines of Rust
- *Easy to test* — No external dependencies

This follows the Unix philosophy: do one thing well, compose with other tools.
