RSS Persistence Implementation Summary
======================================

Branch: feature/rss-persistence
Commits: d03077f, 94e59ef, e1fe5b7

OVERVIEW
--------
Added persistence to the RSS skill so it remembers which feed items have been
seen across daemon restarts. Seen items are stored in the age-encrypted
"rss" namespace using the storage system introduced in cricht-056.

IMPLEMENTATION DETAILS
----------------------

1. Storage Strategy
   - Namespace: "rss"
   - Key: "seen-feeds"
   - Value: Hash-table mapping URL → hash-table of GUID → timestamp
   - Follows token-usage.lisp pattern

2. Lazy Loading
   - State is loaded from storage on first access (via ensure-rss-state-loaded)
   - Prevents unnecessary decryption overhead on daemon startup
   - Thread-safe via *rss-state-lock*

3. Auto-Persistence
   - Enabled by default via *auto-persist-rss* flag
   - Automatically saves state after each mark-seen operation
   - Can be disabled with disable-rss-persistence

4. Data Model
   - Changed from GUID → T to GUID → timestamp
   - Enables future TTL-based cleanup (old items can be expired)
   - Backward compatible (nil timestamps are acceptable)

USAGE EXAMPLES
--------------

;; Check feed (auto-persists seen items)
(rss-check "https://example.com/feed.xml")

;; Get statistics
(rss-state-stats)
;; => (:FEED-COUNT 1 :TOTAL-SEEN-ITEMS 3 :AUTO-PERSIST-P T)

;; Explicit persistence control
(disable-rss-persistence)  ; Turn off auto-save
(mark-seen url items)       ; State in memory only
(persist-rss-state)         ; Explicit save
(enable-rss-persistence)    ; Turn auto-save back on

;; Clear specific feed
(clear-seen "https://example.com/feed.xml")

;; Clear all RSS state
(clear-all-rss-state)
