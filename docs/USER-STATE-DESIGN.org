#+AUTHOR: Gordon J. Broom
#+DATE: 2026-03-02
#+STARTUP: content

/Inspired by OpenClaw's workspace bootstrap pattern, adapted for Crichton's
security-first architecture./

* Overview

Crichton wakes fresh on every session.  These files /are/ its continuity.
They live in =~/.crichton/state/= as plain org-mode files, are loaded at
session startup, and injected verbatim into the LLM system prompt so the
agent always knows who it is, who the user is, and what matters.

#+begin_src ditaa
  +--------------------+
  | ~/.crichton/state/  |
  |                    |
  | SOUL.org           |  <-- persona, values, tone
  | MEMORY.org         |  <-- curated long-term facts
  | USER.org           |  <-- user preferences & context
  |                    |
  | journal/           |
  |   2026-03-02.org   |  <-- today's raw daily log
  |   2026-03-01.org   |  <-- yesterday's daily log
  +--------------------+
#+end_src

** Why plain text?

- Org-mode is the project's documentation standard (per AGENTS.org).
- No schema drift — the LLM reads prose, not data structures.
- Human-editable in Emacs without tooling.
- Encrypted at rest when the user opts in (already have age infrastructure).

** Why not reuse storage/store.lisp?

The KV store is for structured daemon data (meters, RSS state, scheduler
tasks).  User state is /prose for the LLM/ — free-form, human-readable,
injected wholesale into the system prompt.  Different purpose, different
API.

* File Definitions

** SOUL.org — Persona & Values

/Who is Crichton?  How should it behave?/

Loaded on every turn.  Tells the model its identity and behavioural
boundaries.  The user writes this file; the agent may suggest edits but
must ask first.

Example:

#+begin_example
,#+TITLE: Soul

,* Core Truths
- Be genuinely helpful, not performatively helpful.
- Have opinions.  Disagree when warranted.
- Be resourceful before asking.
- You're a guest in someone's life.  Treat access with respect.

,* Boundaries
- Private things stay private.
- When in doubt, ask before acting externally.

,* Vibe
The quietly competent servant who's actually running everything.
#+end_example

Defaults: ship a minimal template on first boot if the file doesn't exist.

** MEMORY.org — Curated Long-Term Memory

/What does Crichton need to remember across sessions?/

Auto-injected in main sessions.  /Not/ injected in channel (Discord)
sessions or subagent contexts — personal facts must not leak.

The agent writes this file when it learns durable facts (user
preferences, ongoing project context, lessons learned).  It is /not/ a
raw log — it is curated.

Example:

#+begin_example
,#+TITLE: Long-Term Memory

,* User Preferences
- Prefers Emacs org-mode over Markdown.
- Uses SBCL for everything.
- Timezone: America/Vancouver (Pacific).

,* Ongoing Projects
- Crichton: secure AI daemon (~/devel/crichton)
- Focus: TUI client, WASM skill pipeline, RSS filtering.

,* Lessons Learned
- Always rebuild client binary after changing package exports.
- Discord heartbeat interval comes from the HELLO payload, not config.
#+end_example

** USER.org — User Context

/Who is the human?  How should the agent address them?/

Loaded on every turn alongside SOUL.org.

Example:

#+begin_example
,#+TITLE: User

,* Identity
- Name: Gord
- Handle: gjbroom
- Location: Victoria, BC, Canada

,* Communication Style
- Prefers concise, direct responses.
- Dislikes filler phrases ("Great question!", "Absolutely!").
- Appreciates dry humour.
#+end_example

** journal/ — Daily Logs

/Raw, append-only daily notes./

Never auto-injected (would blow the context window).  Searchable
on-demand via a =memory= agent tool.  One file per day:
=journal/YYYY-MM-DD.org=.

The agent appends to today's journal when it encounters facts worth
recording but that don't rise to MEMORY.org level.  A pre-compaction
flush writes transient session knowledge here before context is
discarded.

* Architecture

** File Layout

#+begin_src
~/.crichton/state/
├── SOUL.org
├── MEMORY.org
├── USER.org
├── journal/
│   ├── 2026-03-02.org
│   └── ...
#+end_src

Directory permissions: =0700=.  File permissions: =0600=.

** Module: =crichton/state=

New ASDF module: =(:module "state" :components ((:file "bootstrap") (:file "journal")))=.

New package =crichton/state= exporting:

| Function                          | Purpose                                              |
|-----------------------------------+------------------------------------------------------|
| =state-dir=                       | Return/create =~/.crichton/state/= with 0700 perms   |
| =load-bootstrap-files=            | Read SOUL + USER + MEMORY into a plist               |
| =filter-bootstrap-for-session=    | Strip MEMORY for subagent/channel sessions           |
| =bootstrap-system-prompt=         | Build composite system prompt with bootstrap context |
| =journal-dir=                     | Return/create =state/journal/= directory              |
| =journal-today-path=              | Return path for today's journal file                 |
| =journal-append=                  | Append an entry to today's journal                   |
| =journal-search=                  | Search journal files by date range and/or keyword    |
| =append-to-memory=               | Append a curated fact to MEMORY.org                  |
| =estimate-messages-tokens=        | Rough token count estimate for message list          |
| =flush-session-to-journal=        | Pre-compaction dump of session history to journal    |

** Bootstrap Loading Flow

#+begin_src ditaa
  +-------------------+     +--------------------+
  | Agent turn starts  |---->| load-bootstrap-files|
  +-------------------+     +--------+-----------+
                                     |
                             +-------v--------+
                             | Session type?  |
                             +---+--------+---+
                                 |        |
                          main/private  channel/subagent
                                 |        |
                         +-------v--+  +--v---------+
                         | Full set |  | Minimal set|
                         | SOUL     |  | SOUL       |
                         | USER     |  | USER       |
                         | MEMORY   |  +------------+
                         +----+-----+
                              |
                     +--------v---------+
                     | Assemble system  |
                     | prompt with      |
                     | bootstrap context|
                     +--------+---------+
                              |
                     +--------v---------+
                     | *default-system- |
                     |  prompt* +       |
                     | bootstrap text   |
                     +------------------+
#+end_src

The =bootstrap-system-prompt= function concatenates
=*default-system-prompt*= with the loaded bootstrap files, respecting
a configurable character budget (default: 20,000 chars per file,
150,000 total).

** Integration with Agent Loop

=agent/loop.lisp= changes:

1. =run-agent= and =run-agent/stream= gain a =:session-type= keyword
   (default =:main=; alternatives =:channel=, =:subagent=).

2. When =:system= is not explicitly provided, the agent loop calls
   =bootstrap-system-prompt= instead of using
   =*default-system-prompt*= directly:

   #+begin_src lisp
   (let ((system (or system
                     (crichton/state:bootstrap-system-prompt
                      :session-type session-type))))
     ...)
   #+end_src

3. The RPC chat handler passes =:session-type= based on the request
   source (TUI/client → =:main=, Discord → =:channel=).

** Agent Tools for Memory

Two new tools registered in =agent/tools.lisp=:

*** =memory_write=

Write a fact to MEMORY.org or today's journal.

#+begin_src lisp
;; Tool definition
(:name "memory_write"
 :description "Write a durable fact to long-term memory (MEMORY.org)
or today's journal. Use 'memory' target for important, curated facts.
Use 'journal' target for session-specific notes."
 :input-schema (:type "object"
                :properties (:target (:type "string" :enum ("memory" "journal"))
                             :content (:type "string"))
                :required ("target" "content")))
#+end_src

*** =memory_search=

Search journal entries by date range and/or keyword.

#+begin_src lisp
(:name "memory_search"
 :description "Search daily journal entries. Returns matching excerpts."
 :input-schema (:type "object"
                :properties (:query (:type "string")
                             :days-back (:type "integer" :default 7))
                :required ("query")))
#+end_src

* Configuration

New =[state]= section in =config.toml=:

#+begin_src toml
[state]
# Enable/disable bootstrap file injection
enabled = true

# Per-file character limit (prevents huge files from blowing context)
max_file_chars = 20000

# Total character budget for all bootstrap files combined
max_total_chars = 150000

# Encrypt state files at rest (uses existing age identity key)
encrypt = false
#+end_src

* Security Considerations

| Concern                    | Mitigation                                                    |
|----------------------------+---------------------------------------------------------------|
| MEMORY.org leaks to skills | Skills cannot access =state/= — no host function, no RPC op  |
| Channel sessions see personal data | =filter-bootstrap-for-session= strips MEMORY for channels |
| Large files blow context   | Per-file and total character budgets enforced                 |
| State files readable       | =0700= dir, =0600= files; optional age encryption            |
| Journal grows unbounded    | Retention policy: purge journals older than =retention_days=  |

=crichton/state= is orthogonal to sessions (which store message
history) and storage (which stores structured daemon data).  State
files are /prose for the LLM's context window/, not data for
programmatic access.
